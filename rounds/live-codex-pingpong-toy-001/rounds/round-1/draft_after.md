# Toy Approval Adapter Spec

## Purpose

Define a tiny approval adapter contract for smoke testing Whetstone.

## Interface

The adapter MUST expose an HTTP JSON endpoint:

- Method: `POST`
- Path: `/v1/approval`
- Content-Type: `application/json`

Synchronous: the adapter returns a response in the same HTTP call.

## Request

JSON body:

- `request_id` (string, required): caller-provided stable identifier.
- `decision` (string, required): one of the allowed values below.
- `expires_at` (string, required): RFC 3339 timestamp in UTC (`Z`), e.g. `2026-05-02T12:34:56Z`.

### Allowed `decision` values

- `approve`
- `reject`

`decision` validation is case-sensitive; any other value is invalid.

## Response

### Success

If the request is accepted, respond:

- Status: `200`
- JSON body:
  - `request_id` (string): echoes the request.
  - `decision` (string): echoes the normalized decision (exactly `approve` or `reject`).
  - `submitted_at` (string): RFC 3339 timestamp in UTC (`Z`), generated by the adapter at acceptance time.

### Error

If validation fails, the adapter MUST reject the request:

- Status: `400`
- JSON body:
  - `error` (object):
    - `code` (string): one of:
      - `invalid_decision`
      - `invalid_expires_at`
      - `expired_request`
      - `missing_field`
    - `message` (string): human-readable.

## Validation

- Missing required fields: reject with `400` and `error.code=missing_field`.
- Invalid decisions: reject with `400` and `error.code=invalid_decision`.
- `expires_at` parsing: must be RFC 3339 with UTC `Z`; otherwise reject with `400` and `error.code=invalid_expires_at`.
- Expiration: if the adapter's current UTC time is greater than or equal to `expires_at`, reject with `400` and `error.code=expired_request`.
